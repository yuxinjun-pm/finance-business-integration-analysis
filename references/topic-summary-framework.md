# Topic Summary Framework

Use this reference when the user provides finance-business courseware, lecture slides, scanned notes, or PDFs and asks you to digest the material.

## Objective

Turn a pile of course materials into:
- a clear knowledge backbone
- a set of reusable concepts and formulas
- a compact framework the user can apply later

## Required output layers

### Layer 1: Course main line
Answer:
- What is the course really about?
- What management problem is it trying to solve?
- What is the core decision logic behind the topic?

### Layer 2: Session-by-session compression
For each session:
- title or inferred theme
- one-sentence core conclusion
- the 2-4 most important concepts
- any critical formula, distinction, or case insight

### Layer 3: Cross-session synthesis
After summarizing individual sessions, compress them into:
- one integrated logic chain
- the concepts most likely to be confused
- the reusable question set the user can apply to future cases

## Default structure

Use this order unless the user asks for another format:

1. Main line of the course
2. Core conclusions by session
3. Unified conceptual framework
4. Most important formulas or distinctions
5. Practical questions to ask in future analysis

## Compression rules

- Do not summarize page by page by default.
- Do not restate teacher biography, grading, or administrative content unless relevant.
- Preserve important distinctions, such as:
  - fixed vs variable cost
  - relevant vs irrelevant cost
  - contribution profit vs accounting net profit
  - structural vs execution cost drivers
  - short-term decision vs long-term design decision

## Good summary test

The summary is good if the user can answer:
- What is the central management logic?
- What are the few formulas or distinctions that matter most?
- How would I apply this to a new business problem?
