# Output Templates

Use this reference to match the response style to the user's actual need.

## 1. Classroom speaking answer

Use when the user likely needs a concise spoken answer.

Structure:
1. state the core conclusion
2. give the 2-4 key supporting points
3. end with the management implication

Target style:
- short
- direct
- easy to speak aloud

## 2. Assignment-ready structured answer

Use when the user needs a written case answer.

Structure:
1. analysis objective and metric definition
2. problem 1: calculations and conclusion
3. problem 2: comparison and driver diagnosis
4. problem 3: improvement actions
5. problem 4: algorithm or framework design
6. integrated conclusion

Target style:
- complete
- structured
- explicit formulas and assumptions

## 3. Standard answer

Use when the user wants a higher-standard reference answer.

Must include:
- clear profit lens
- total result
- segment comparison
- specific drivers
- quantified actions
- implementation implications

## 4. Review postmortem

Use when the user asks to compare your answer with a final delivery.

Structure:
1. overall conclusion
2. where my answer was right
3. where the final delivery was stronger
4. root causes of the gap
5. future upgrade checklist

## 5. Practical memo

Use when the user wants management implications rather than academic structure.

Structure:
1. situation
2. key finding
3. implication
4. action recommendation
5. follow-up mechanism

## Style rule across all templates

Whatever the format:
- anchor all major claims to evidence
- separate measurement from interpretation
- separate interpretation from recommendation
- make actions match the diagnosed driver
